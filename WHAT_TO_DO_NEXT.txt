1. Choose where to host it
==========================
Pick one production hosting path.

Recommended easiest options:
- Render
- Railway
- Fly.io
- Azure App Service
- DigitalOcean App Platform

You need two services:
1. API service for the FastAPI backend.
2. Static web service for the /web frontend.

The API start command should be:

python -m uvicorn grocery_optimizer.api.app:app --host 0.0.0.0 --port $PORT --app-dir src

If your host does not provide $PORT, use the port they document.


2. Create a production PostgreSQL database
==========================================
Do not use SQLite for real production traffic.

Create a managed PostgreSQL database using your host or a database provider such as:
- Neon
- Supabase
- Railway Postgres
- Render Postgres
- Azure Database for PostgreSQL

After creating it, copy the connection string. It should look similar to:

postgresql://USER:PASSWORD@HOST:5432/DATABASE

Save that value for the GROCERY_DATABASE_URL environment variable.

Also enable automatic backups or point-in-time recovery in the database provider dashboard.


3. Set production environment variables
=======================================
Set these on the hosted API service:

GROCERY_ENV=production
GROCERY_DATABASE_URL=postgresql://USER:PASSWORD@HOST:5432/DATABASE
GROCERY_MANAGED_BACKUPS_ENABLED=true
GROCERY_API_CORS_ORIGINS=https://YOUR-FRONTEND-DOMAIN.com
GROCERY_PUBLIC_BASE_URL=https://YOUR-API-DOMAIN.com
GROCERY_ADMIN_TOKEN=MAKE_A_LONG_RANDOM_SECRET
GROCERY_RATE_LIMIT_ENABLED=true
GROCERY_TOKEN_TTL_MINUTES=10080
GROCERY_ROUTE_COST_PER_KM=0.70
GROCERY_ROUTE_MIN_NET_SAVINGS=1.50
LIVE_PRICING_FLIPP_KEY=
LIVE_PRICING_RETAILER_CATALOG_KEY=
LIVE_PRICING_LOBLAW_KEY=
LIVE_PRICING_METRO_KEY=
LIVE_PRICING_IGA_KEY=
LIVE_PRICING_FLYER_OCR_KEY=

Generate GROCERY_ADMIN_TOKEN as a long random string. Do not reuse a password.

Example PowerShell command to generate one locally:

[Convert]::ToBase64String((1..48 | ForEach-Object { Get-Random -Maximum 256 }))

5. Add real grocery pricing provider credentials
================================================
The app has provider adapters, but you need real partner/API access for live prices.

Start with one of these:
- Loblaw/PC Express partner, affiliate, or controlled proxy feed for Maxi, Provigo, and T&T.
- Metro/Super C partner, affiliate, or controlled proxy feed.
- IGA/Voila partner, affiliate, or controlled proxy feed.
- Flipp/RapidAPI grocery or flyer API only if its license/terms are acceptable.
- A grocery affiliate/partner product feed.
- A public flyer OCR pipeline for Supermarche PA, Akhavan, and similar ethnic-value stores.

Set the relevant API key as an environment variable:

LIVE_PRICING_FLIPP_KEY=YOUR_KEY

or:

LIVE_PRICING_RETAILER_CATALOG_KEY=YOUR_KEY

Spreadsheet-specific optional keys:

LIVE_PRICING_LOBLAW_KEY=YOUR_KEY
LIVE_PRICING_METRO_KEY=YOUR_KEY
LIVE_PRICING_IGA_KEY=YOUR_KEY
LIVE_PRICING_FLYER_OCR_KEY=YOUR_KEY

Then edit:

config/live_pricing/providers.json

Enable the provider you are using and replace placeholder fields such as:
- base_url
- headers
- result_list_path
- item_name_path
- store_chain_path
- price_path
- currency_path

After deployment, test:

GET https://YOUR-API-DOMAIN.com/pricing/providers

You want your provider to show:

configured: true

Do not enable undocumented or ToS-gray scraping routes until you have reviewed legal/terms risk.

6. Verify production readiness
==============================
After the API is deployed, open:

https://YOUR-API-DOMAIN.com/health
https://YOUR-API-DOMAIN.com/ready

Then check deployment status using your admin token:

PowerShell:

$headers = @{ Authorization = "Bearer YOUR_GROCERY_ADMIN_TOKEN" }
Invoke-RestMethod -Headers $headers https://YOUR-API-DOMAIN.com/deployment/status

The response should show:

ready: true
errors: []

Warnings are acceptable during early beta, but errors need to be fixed before launch.

===============================

Before pushing future code changes, run:
